## 1. Исходный запрос

> Оригинальная формулировка задачи от пользователя.

```text
Еще, если мы можем сохранять настройки, вместо copy json сделай возможность загрузки json.
```

---

## 2. Цель

Сделать workflow JSON-конфига в playground двусторонним: оставить сохранение в файл и заменить `Copy JSON` на загрузку JSON-файла обратно в форму playground.

---

## 3. Бизнес-ценность

Сохранение без обратной загрузки делает конфиг неполезным в реальной работе. Поддержка `Save JSON` + `Load JSON` позволяет повторно использовать настройки playground без ручного переноса значений.

---

## 4. Целевые пользователи

| Пользователь / Роль | Как использует результат                                      |
| ------------------- | ------------------------------------------------------------- |
| Разработчик         | Сохраняет и повторно загружает настройки preview/runtime      |
| Дизайнер            | Возвращается к ранее подобранным параметрам без ручного ввода |

---

## 5. Сценарии использования

1. Пользователь сохраняет JSON-конфиг из playground в файл.
2. Позже пользователь загружает этот JSON обратно и получает те же значения формы.
3. При ошибке файла playground показывает понятное сообщение и не ломает текущее состояние.

---

## 6. Ограничения

| Тип ограничения | Описание                                                         |
| --------------- | ---------------------------------------------------------------- |
| Scope           | Меняется только workflow playground, не public runtime API       |
| UX              | `Copy JSON` удаляется и заменяется на `Load JSON`                |
| Config          | Background preview по-прежнему не участвует в JSON import/export |
| Compatibility   | Загружаться должен сохраненный playground JSON текущего формата  |

---

## 7. Риски

| Риск                                           | Вероятность | Влияние | Митигация                                                               |
| ---------------------------------------------- | ----------- | ------- | ----------------------------------------------------------------------- |
| Загрузка невалидного JSON ломает форму         | Medium      | High    | Добавить безопасный parse и валидацию полей перед применением           |
| Import меняет лишний state playground          | Medium      | Medium  | Применять только canonical config fields, не трогать background preview |
| UI загрузки выглядит инородно рядом с кнопками | Low         | Low     | Стилизовать `Load JSON` как control того же ряда                        |

---

## 8. Варианты реализации

### Вариант A: Оставить textarea как источник импорта

- Описание: пользователь вручную вставляет JSON в textarea.
- Плюсы: минимум UI.
- Минусы: медленно, неудобно, не соответствует запросу на загрузку файла.

### Вариант B: File-based JSON import

- Описание: заменить `Copy JSON` на `Load JSON` с выбором локального файла и применением конфига.
- Плюсы: естественный round-trip workflow и минимальный риск ошибок ручного копирования.
- Минусы: нужно добавить parser/validation и file input в UI.

**Рекомендуемый вариант:** Вариант B

## 10. Допущения

- Загружается JSON того же формата, который экспортирует playground.
- При импорте допустимо менять только form state, связанный с `PlaygroundConfig`.
- Если ассет не загружен, значения формы все равно применяются, а preview ведет себя как сейчас.

---

## 11. Критерии успеха

| Критерий                                          | Как измеряется                                                  |
| ------------------------------------------------- | --------------------------------------------------------------- |
| JSON workflow стал двусторонним                   | В playground доступны `Save JSON` и `Load JSON`                 |
| Конфиг можно повторно применить без ручного ввода | Значения формы обновляются после загрузки файла                 |
| Ошибки импорта контролируемы                      | Невалидный JSON показывает понятное сообщение в preview message |

---

## 12. Итоговое видение задачи

В секции `Config` playground вместо `Copy JSON` появляется `Load JSON`. Пользователь может выбрать локальный JSON-файл, playground безопасно читает и применяет canonical config-поля к форме и runtime. Background preview и прочий dev-only state остаются вне JSON import/export.
