## 1. Исходный запрос

> Оригинальная формулировка задачи от пользователя.

```text
Нужно сделать задачу по изменению задника в playground. Варианты смены палитры из предложенных вариантов + возможность подгрузить картинку в качестве background.
```

---

## 2. Цель

Добавить в playground dev-only управление background preview, чтобы пользователь мог быстро проверять спрайт на артефакты, прозрачность и читаемость на разных типах задника без изменения runtime-конфига анимации и без участия background в JSON-экспорте.

---

## 3. Бизнес-ценность

Playground используется как инструмент визуальной валидации. Возможность переключать задник позволяет раньше замечать дефекты ассетов и лучше оценивать качество спрайта в разных условиях, не вынося эту проверку во внешний редактор или отдельные тестовые страницы.

---

## 4. Целевые пользователи

| Пользователь / Роль | Как использует результат                                                             |
| ------------------- | ------------------------------------------------------------------------------------ |
| Разработчик         | Проверяет контраст, прозрачность и артефакты спрайта в preview-сцене                 |
| Дизайнер            | Быстро оценивает качество ассета на разных фонах без изменения анимационного конфига |

---

## 5. Сценарии использования

1. Пользователь переключает несколько готовых палитр и оценивает края/прозрачность спрайта.
2. Пользователь загружает локальную background-картинку и проверяет спрайт на референсном окружении.
3. Пользователь меняет режим размещения картинки (`cover`, `contain`, `repeat`, `stretch`) под нужный тип проверки.

---

## 6. Ограничения

| Тип ограничения | Описание                                                                          |
| --------------- | --------------------------------------------------------------------------------- |
| Scope           | Изменение относится только к playground и не меняет public runtime API библиотеки |
| Persistence     | Background живет только в текущей сессии playground                               |
| Config          | Background не участвует в JSON-конфиге и не должен экспортироваться/копироваться  |
| UX              | Палитры подбирает агент, обязательные варианты: белый, черный, яркий зеленый      |

---

## 7. Риски

| Риск                                         | Вероятность | Влияние | Митигация                                                                            |
| -------------------------------------------- | ----------- | ------- | ------------------------------------------------------------------------------------ |
| Смешение dev-preview state и runtime-конфига | Medium      | High    | Явно разделить state background и `PlaygroundConfig`, зафиксировать это в docs и ADR |
| Перегрузка sidebar лишними control-ами       | Low         | Medium  | Оставить компактную отдельную секцию `Background` с минимальным набором полей        |
| Неочевидное поведение background image       | Medium      | Medium  | Явно ограничить режимы до `cover`, `contain`, `repeat`, `stretch`                    |

---

## 8. Варианты реализации

### Вариант A: Только палитры

- Описание: добавить только пресеты цветов без загрузки картинки.
- Плюсы: минимальная сложность.
- Минусы: не покрывает запрос пользователя и реальные сценарии проверки.

### Вариант B: Палитры + локальная картинка + selectable image mode

- Описание: добавить цветовые пресеты, локальную загрузку background image и выбор режима отображения.
- Плюсы: полностью покрывает задачу, остается в рамках dev-only preview.
- Минусы: чуть больше UI и локального состояния в playground.

**Рекомендуемый вариант:** Вариант B

## 10. Допущения

- Background нужен только для visual QA ассета.
- При отсутствии background image playground использует цветовую палитру как fallback.
- Изменение не требует расширения `PlaygroundConfig` и не меняет формат экспортируемого JSON.

---

## 11. Критерии успеха

| Критерий                                                   | Как измеряется                                                      |
| ---------------------------------------------------------- | ------------------------------------------------------------------- |
| Пользователь может быстро проверять спрайт на разных фонах | В playground доступны palette presets и загрузка локальной картинки |
| Background не загрязняет анимационный конфиг               | JSON export/copy не меняется и не содержит background-полей         |
| Поведение картинки предсказуемо                            | Для background image доступен явный select с режимами отображения   |

---

## 12. Итоговое видение задачи

Playground получает отдельную секцию `Background`, которая управляет только preview-сценой. Пользователь может выбрать одну из нескольких палитр или загрузить локальную картинку и переключать режим ее отображения. Эти настройки предназначены исключительно для визуальной проверки качества спрайта, сохраняются только в текущей сессии и не входят в JSON-конфиг анимации.
