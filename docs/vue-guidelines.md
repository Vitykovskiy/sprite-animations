# Vue Guidelines

## Component Style

- Использовать `script setup` в новом коде.
- Один компонент отвечает за один UI-блок или экран.
- Не использовать Options API без отдельного обоснования.
- Не использовать class-based Vue patterns.

## Composition API

- `ref` для примитивов и DOM refs.
- `reactive` для компактных связанных групп состояния.
- `computed` только для производных значений.
- `watch` только для side effects и синхронизации с внешним миром.

## Template Rules

- Не писать нетривиальные вычисления inline.
- Не вызывать функции с побочными эффектами из template bindings.
- Для классов использовать короткие и очевидные биндинги.
- Повторяющиеся блоки выносить в компонент только если это реально уменьшает сложность.

## Composable Rules

- Сложная UI-логика и orchestration выносятся в composable.
- Composable описывает поведение, компонент в основном описывает разметку.
- Composable не должен использовать `document.getElementById`.
- Composable не должен импортировать `.vue` файлы.
