# 01 — Product Vision

> Заполняется агентом после качественного снятия задачи.

---

## Проблема

<!-- Какую проблему решает этот продукт? Почему она важна? -->
Команде нужна собственная библиотека для 2D sprite-анимации в вебе, чтобы полностью контролировать загрузку ассетов, тайминг, позиционирование и поведение анимации на `canvas` без зависимости от чужих абстракций.

---

## Ценность

<!-- Какую ценность получает пользователь/бизнес? -->
Библиотека сокращает время на повторяющуюся реализацию sprite-анимаций, дает предсказуемый runtime и единый внутренний способ работы разработчика и дизайнера с ассетами и параметрами анимации.

---

## Целевые пользователи

| Сегмент        | Описание                     | Ключевая потребность     |
|----------------|------------------------------|--------------------------|
| Разработчик    | Подключает библиотеку в веб-проект | Управлять sprite-анимацией и отрисовкой на `canvas` через простой API |
| Дизайнер       | Подготавливает и проверяет ассеты | Быстро валидировать сетку кадров, тайминг и позиционирование через playground |

---

## High-level Use Cases

1. Проиграть анимацию персонажа из регулярного sprite sheet в canvas-игре.
2. Подобрать параметры сетки, масштаба, позиции и тайминга в playground.
3. Сохранить конфиг анимации для повторного использования в проекте.

---

## Ограничения

- Фокус первой версии ограничен вебом и `canvas`.
- В MVP нет привязки к `React`, `Vue` или другим фреймворкам.
- В MVP нет визуального редактора анимаций, state machine и поддержки отдельных файлов кадров.

---

## Критерии успеха

| Критерий               | Метрика / Индикатор         |
|------------------------|-----------------------------|
| Библиотека закрывает базовый внутренний use case | Персонаж из sprite sheet корректно анимируется в `canvas` |
| Playground полезен команде | Через playground можно проверить сетку кадров, позицию и тайминг без ручной правки кода |
| API остается простым | Базовый сценарий использования покрывается без фреймворк-оберток и сложной state-модели |
