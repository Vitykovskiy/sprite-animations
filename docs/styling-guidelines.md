# Styling Guidelines

## CSS Rules

- Один frontend surface может использовать общий `styles.css`, пока это остается обозримым.
- Токены цветов, радиусов и теней хранить в CSS custom properties.
- Не смешивать layout fixes и component skinning без необходимости.
- Новые классы называть по смыслу, а не по визуальному эффекту.
- Не завязывать CSS на autogenerated Vue attributes.

## Tooling

- Форматирование кода: Prettier
- Code style / anti-patterns: ESLint
- CSS linting: Stylelint

## Review Checklist

- Нет случайного drift форматирования
- Нет новых CSS anti-patterns, которые не проходят Stylelint
- Новые классы и токены читаемы и семантичны
